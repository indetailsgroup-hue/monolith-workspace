ผังบังคับบัญชา (จาก ผู้บังคับบัญชา/ผู้ใต้บังคับบัญชา ในทุก JD)

กรรมการผู้จัดการ (MD) ← จัดซื้อต้องนำเสนอ MD อนุมัติใบสั่งซื้อ
└─ ผู้จัดการทั่วไป (GM)
   ├─ ผู้จัดการโรงงาน (Plant Manager) — ฝ่ายผลิตและงานคุณภาพ
   │   ├─ หัวหน้าฝ่ายผลิต (Production Head) ─ เจ้าหน้าที่ฝ่ายผลิต
   │   ├─ หัวหน้าฝ่ายซ่อมบำรุง (Maintenance Head)
   │   ├─ เจ้าหน้าที่/หัวหน้าฝ่ายประกันคุณภาพ (QA — ขึ้นตรงผู้จัดการโรงงาน)
   │   ├─ หัวหน้าฝ่ายความปลอดภัย
   │   ├─ หัวหน้าฝ่ายคลังสินค้า ─ เจ้าหน้าที่คลังสินค้า (Warehouse)
   │   ├─ หัวหน้าฝ่ายโลจิสติกส์ — จัดส่ง "เจ้าหน้าที่ติดตั้ง + ชิ้นงาน" ไปหน้างาน
   │   └─ หัวหน้าฝ่าย R&D
   ├─ ผู้จัดการโครงการ (Project Manager) — สังกัด "ฝ่ายงานติดตั้ง" = เจ้าของเส้นส่งมอบ
   │   ├─ ที่ปรึกษา/ผู้ช่วย PM (Assistant PM) — คุมหัวหน้าทีมติดตั้ง
   │   ├─ หัวหน้าฝ่ายติดตั้ง (Installation Dept Head)
   │   │   └─ หัวหน้าทีมติดตั้ง (Team Lead — ต่อทีม/บ้าน)
   │   │       └─ หัวหน้าช่างติดตั้ง (Chief Technician)
   │   │           ├─ ช่างติดตั้ง (Technician)
   │   │           └─ ผู้ช่วยช่างติดตั้ง (Assistant Technician)
   │   └─ หัวหน้าฝ่ายวางแผนการผลิต
   │       ├─ เจ้าหน้าที่วางแผนการผลิต (Production Planning Staff — AUTO CAD, ถอดแบบ/วัสดุ, ประเมินราคา)
   │       └─ เจ้าหน้าที่เขียนแบบ (Draftsman — ไฟล์งานตัด/งานเจาะ → ฝ่ายผลิต)
   ├─ ผู้จัดการฝ่ายบัญชีและการเงิน
   │   ├─ เจ้าหน้าที่บัญชี (AP/AR/ต้นทุน — ภพ.30, ภงด.1/3/53, สปส.10-1, รายงานอายุหนี้)
   │   ├─ เจ้าหน้าที่การเงิน (invoice/ใบกำกับ, เช็ค/โอน, petty cash)
   │   └─ เจ้าหน้าที่จัดซื้อ (Purchasing — **สังกัดฝ่ายบัญชีการเงิน**; PR→เสนอ MD อนุมัติ→PO, ประเมิน Supplier)
   ├─ ผู้จัดการฝ่ายบุคคล (HR Manager) ─ เจ้าหน้าที่ HR
   ├─ ผู้จัดการฝ่ายการตลาด (JD ยังไม่มีในชุด)
   ├─ ผู้จัดการฝ่ายขาย (Sales Manager) ─ เจ้าหน้าที่ฝ่ายขาย (เก็บ requirement, ใบเสนอราคา, **ตอบแชทลูกค้า**, after-sales)
   └─ ผู้จัดการฝ่ายออกแบบ (Designer Manager) ─ Designer (พบลูกค้า, 3D → ลูกค้า/ฝ่ายวางแผนฯ, Floorplan → ฝ่ายติดตั้ง)

ข้อเท็จจริงสำคัญต่อระบบ (ยกจาก JD ตรง ๆ)

เรื่อง JD ระบุ ผลต่อระบบ
สายติดตั้ง 4 ชั้น หัวหน้าฝ่าย → หัวหน้าทีม → หัวหน้าช่าง → ช่าง+ผู้ช่วย "หัวหน้างาน 1 คน/บ้าน" ในโมเดล installation-pm = หัวหน้าทีมติดตั้ง; approver "หัวหน้าทีม Installation" ใน workflow spec ตรงกับ JD ตรง ๆ
PM คุมทั้งติดตั้ง + วางแผนการผลิต subordinates: หัวหน้างานติดตั้ง + หัวหน้าฝ่ายวางแผนการผลิต escalation งานติดตั้ง → PM (ตรง workflow spec: timeout → project_manager); Draftsman/PP staff ขึ้น PM ไม่ใช่โรงงาน
ช่าง/ผู้ช่วย/หัวหน้าทุกระดับ: "ส่งรายงานการติดตั้งให้ผู้บังคับบัญชาทุกครั้ง" + KPI "รายงานประจำวัน"; หัวหน้า: "สรุปรายงานผลทุกวัน" + ควบคุมเวลาเข้า-ออกไซต์ หน้าที่ตามใบงานจริง field_reports รายวัน = ข้อผูกพันตาม JD ไม่ใช่ฟีเจอร์เสริม; เวลาเข้า-ออกไซต์ควรเก็บใน daily report
Purchasing: PR → เสนอกรรมการผู้จัดการอนุมัติ → PO อำนาจอนุมัติการสั่งซื้อ = MD approval chain จัดซื้อในระบบต้องจบที่ MD ไม่ใช่ GM/ผู้จัดการฝ่าย
โลจิสติกส์ (ใต้โรงงาน) จัดส่ง "ช่างติดตั้ง + ชิ้นงาน" JD Factory Manager การนัดส่งของ+ทีมเข้าหน้างาน = จุดเชื่อม Factory↔Installation ที่ระบบต้องมี (packet + คน + รถ)
QA เป็นด่านสุดท้ายก่อนส่งมอบ, อนุมัติของดี/ของเสียได้เอง, คุมเครื่องมือวัด+สอบเทียบ JD QA gate คุณภาพก่อนส่งมอบ → ตรงกับ capture verify / trust chain มุมโรงงาน
Sales: "ตอบแชทให้ข้อมูลลูกค้า ติดตามสินค้า รับเรื่องร้องเรียน" + บริการหลังการขาย JD Sales Staff ยืนยันบทบาท Sale ใน LINE OA (ผู้ตอบหลักของกลุ่ม customer) + after-sales อยู่กับ Sale
Designer ส่งมอบ 3 ทาง: 3D→ลูกค้า, 3D→ฝ่ายวางแผนฯ, Floorplan→ฝ่ายติดตั้ง JD Designer/Manager เส้นทางแบบใน data model: design → PP → installation ตรงตามนี้
Warehouse: เบิก-จ่ายทั้งงานผลิตและงานติดตั้ง, FIFO, NC Product, รายงานสต๊อกรายวัน/เดือน JD Warehouse คลังแตะโปรเจกต์ติดตั้งด้วย (เบิกอุปกรณ์ให้ทีมช่าง) — เพิ่มใน access matrix
