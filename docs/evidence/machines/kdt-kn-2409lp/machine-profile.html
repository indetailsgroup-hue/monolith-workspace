3. รูปแบบไฟล์ / เส้นทางไฟล์ KDT (File format & KDT path)

ฟอร์แมตไฟล์หลักที่เครื่องรัน authoritative_native_format
G-code ไฟล์ .nc (ISO/RS-274) — คอนโทรลรับ .nc / .cnc / .txt; ฟอร์แมตรันตัวที่สองคือ HP-PLT (.plt) ยืนยันแล้ว
ไฟล์ที่ผมส่งเข้าเครื่องจริง ๆ คือ .nc G-code — นี่คือ native format ที่ NCstudio เอาไปรันโดยตรง
ฟอร์แมตที่นำเข้าได้ accepted_import_formats
.nc, .cnc, .txt (G-code), .plt (HP-PLT), และ DXF (import แล้วแปลง) ตามเอกสาร
DXF เข้าได้แบบนำเข้า-แปลง ไม่ใช่รันตรง ส่วน nesting shell รุ่นใหม่อาจรับ layout เพิ่ม — ต้องดูจากหน้า import จริง
เส้นทาง/โฟลเดอร์ไฟล์บนเครื่อง (KDT path) program_path
ยังไม่รู้ค่าจริง — ตามแบบ NCstudio ไฟล์โปรแกรมมักอยู่ในโฟลเดอร์งานเฉพาะแล้วเปิดผ่านเมนู Open/Load; เส้นทางเป๊ะ (เช่น D:\...\ หรือโฟลเดอร์ nesting ของ WoodSystem) ต้องเปิดเครื่องจด ต้องเช็กเครื่อง
"KDT path" นี้ผมตีความว่าเป็นเส้นทางที่เครื่องดึงไฟล์ .nc ไปรัน — ผมยังไม่ได้ยืนยันโฟลเดอร์จริง ถ้าคุณหมายถึง "เส้นทางส่งงานจาก CAM → เครื่อง" (pipeline) บอกได้ เดี๋ยวแยกให้
วิธีส่งไฟล์เข้าเครื่อง delivery_channel
USB flash หรือ network share → เปิดใน NCstudio ต้องเช็กเครื่อง
ชื่อไฟล์ / encoding / line-ending filename_encoding
นามสกุล .nc; แนะนำ ASCII, ชื่อสั้น, เลี่ยงอักขระไทย/เว้นวรรค; ข้อความ ASCII + CRLF (Windows) ต้องเช็กเครื่อง
งานวิจัยไม่ยืนยันข้อจำกัดความยาว/อักขระเป๊ะ — ต้องทดสอบไฟล์จริงกับเครื่อง
