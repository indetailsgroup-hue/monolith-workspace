สรุปผู้บริหาร (Executive Summary)

เครื่อง KDT KN-2409 เป็นเครื่อง CNC nesting router แบบ flat-bed จริง ผลิตโดย KDT (Guangdong Nanxing, kdtmac.com) รุ่นในตระกูล "2 Series" ที่เผยแพร่คือ KN-2409NL / KN-2409DL / KN-2409L ระบบควบคุมคือ Weihong NCstudio (ซอฟต์แวร์คอนโทรลบน PC) โดยมี WoodSystem เป็น HMI shell เฉพาะงานไม้วางทับ NCstudio อีกชั้น

ประเด็นสำคัญที่รายงานนี้เคลียร์ได้: (1) ข้อขัดแย้ง Y envelope 1220 vs 1260 mm สรุปแล้วว่า 1220 mm คือค่าจริง (1250–1260 คือขนาด table/bed ในโบรชัวร์ ไม่ใช่ระยะวิ่ง); (2) รุ่นยืนยันเป็น KDT KN-2409LP (เจ้าของยืนยัน; แคตตาล็อกสาธารณะมีแต่ NL/DL/L → LP เป็นรหัสรุ่นย่อย/ออปชัน); (3) วางกรอบ Delivery Contract จาก native format จริงของ NCstudio (G-code .nc) ได้แล้ว แต่ค่าที่ผูกกับตัวเครื่อง (tool table, work origin, เวอร์ชันจริง, วิธีส่งไฟล์) ต้องยืนยันหน้าเครื่อง ก่อนใช้จริง

C. Capability Assessment & การเคลียร์ข้อขัดแย้ง

ข้อขัดแย้ง #1 — Y envelope 1220 vs 1260 mm RESOLVED

ค่าจริง = 2850 (X) × 1220 (Y) mm คือ working area / travel envelope ที่ใช้งานได้จริง ส่วน ~1250–1260 mm คือขนาด table/bed (nominal) ที่ปรากฏใน marketing copy ไม่ใช่ระยะวิ่ง — ยืนยันตรงกันจาก 4 แหล่ง KDT/ดีลเลอร์อิสระ (byte-identical "2850 × 1220 mm") และ 1220 mm พอดีกับความกว้างแผ่น 4 ฟุต (คลาส 8×4 ตามรหัส "2409")
→ ให้บันทึกใน machine profile เป็น working_area_x=2850, working_area_y=1220 และทิ้งค่า 1260 (เป็น bed size เท่านั้น)

ข้อขัดแย้ง #2 — suffix "LP" CONFIRMED BY OWNER

เจ้าของเครื่องยืนยันรุ่นคือ KDT KN-2409LP (owner attestation). หมายเหตุ: แคตตาล็อก KDT สาธารณะระบุเป็น NL / DL / L ไม่มี LP — ดังนั้น "LP" น่าจะเป็นรหัสรุ่นย่อย/ออปชัน/สเปกเฉพาะภูมิภาค (อาจ = Low-Pressure vacuum) ยังแนะนำให้จดรหัสเต็ม + serial จาก nameplate (checklist ข้อ 1) ไว้อ้างอิงเวลาสั่งอะไหล่/ซัพพอร์ต

เหตุผลที่คง status = NOT_ASSESSED / release = PROHIBITED: ไม่มีค่าที่ยืนยันจากเครื่องจริงเลยแม้แต่ค่าเดียว — เวอร์ชันคอนโทรลเลอร์/HMI, tool table, work origin, ช่องส่งไฟล์ ล้วนเป็นค่าอนุมานจากเอกสาร การปล่อยงานผลิตต้องผ่าน gate ยืนยันครบก่อน

D. เช็กลิสต์ยืนยันหน้าเครื่อง (6 ข้อ)

ทั้ง 6 ข้อนี้ ทำได้เฉพาะที่ตัวเครื่องจริงเท่านั้น — Deep research ทางเว็บทำแทนไม่ได้ ผลของแต่ละข้อคือหลักฐานที่จะยกสถานะ assessment

  1. ยืนยัน suffix "LP" ที่ nameplate — ถ่ายรูปป้ายรุ่น/serial plate อ่านรหัสรุ่นเต็ม (KN-2409__) + serial no. เทียบกับรุ่นที่ KDT เผยแพร่ (NL/DL/L) ว่า LP คือรุ่นจริง/ตัวเลือกใด — บันทึกลง profile
  2. NCstudio & WoodSystem — About / Version — เปิดหน้า Help > About บันทึก เวอร์ชัน NCstudio (V?), เวอร์ชัน WoodSystem shell + build, และรุ่นการ์ดคอนโทรล (Lambda 4S/5S / NK300CX) ถ้ามีแสดง
  3. Known-good job (immutable) — สร้างงานอ้างอิงมาตรฐาน 1 ชิ้น (ชิ้นง่าย มี feature เจาะ+ตัด+ทิศทาง) แล้ว ล็อกเป็น golden reference (archive, checksum/แฮช, ห้ามแก้) ใช้เป็นตัวเทียบทุกครั้งที่เปลี่ยนโพสต์/เวอร์ชัน
  4. หน้าจอ Import — เก็บ file extensions จริง — ถ่ายหน้าจอ dialog เปิด/นำเข้าไฟล์ ให้เห็น ตัวกรองนามสกุลจริงที่เครื่องรับ (.nc/.cnc/.txt/.plt/.dxf/...) เพื่อ freeze accepted_import_formats
  5. Tool table & Work origin — ส่งออก/ถ่ายภาพ tool table: การแมป T1..Tn → ตำแหน่งแม็กกาซีนจริง, ค่า tool length/offset, และวิธีเรียกเปลี่ยนทูล (M6 หรือ bare T). บันทึก WCS G54 offset + มุม origin จริงของแผ่น (ล่างซ้าย?) และทิศแกน
  6. วิธีส่งไฟล์ + physical mapping (ปกปิด secret) — บันทึกช่องทางส่งไฟล์จริง (USB/เน็ตเวิร์ก/แชร์) และทดสอบ physical mapping: ยืนยันว่าพิกัดในไฟล์แมปกับตำแหน่งจริงบนโต๊ะถูกต้อง (origin/ทิศแกน) ผ่าน dry-run/air-cut — โดยเวลาบันทึกเอกสารให้ปิดบัง/redact ความลับ (รหัสผ่าน, license key, พารามิเตอร์ลับของผู้สร้างเครื่อง) ไม่ให้หลุดออกนอกเอกสารควบคุม
หมายเหตุข้อ 6: ตีความ "physical mapping ที่ปิดบัง secret" ว่า = การยืนยันการแมปพิกัด→ตำแหน่งจริง โดยปกปิดข้อมูลลับในเอกสาร ถ้าคุณหมายถึงอย่างอื่น (เช่น ฟีเจอร์/คีย์เฉพาะของ WoodSystem) บอกได้ เดี๋ยวปรับให้ตรง
