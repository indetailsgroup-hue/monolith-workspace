Fleet Machine Onboarding Kit

โปรไฟล์ชั่วคราวจากเอกสาร/วิจัย → เดินงานได้ก่อน → วิศวกรเซ็ต+ยืนยันหน้าเครื่อง → ปล่อยผลิตจริง · ขยายได้หลายเครื่อง
1. กระบวนการ 2 เฟส 2. นโยบายชั่วคราว P0/P1 3. Template โปรไฟล์ 4. Fleet Register 5. เช็กลิสต์วิศวกร 6. เพิ่มเครื่องใหม่
หลักการความปลอดภัยที่ยึดตลอด: โปรไฟล์ที่สร้างจากเอกสาร/วิจัยยัง ไม่ยืนยันหน้าเครื่อง — ค่าต่าง ๆ (envelope, tool table, origin, เวอร์ชัน, path) ถือว่า "ยังไม่จริง" จนกว่าวิศวกรจะเซ็ตและเซ็นรับรองหน้าเครื่อง ห้ามปล่อยงานผลิตจริงแบบไม่มีคนคุมจนกว่าจะผ่านเฟส 2

1. กระบวนการ 2 เฟส (Provisional → Verified)

A · Deep-Research
ค้นเอกสาร/สเปก/manual สร้างโปรไฟล์ชั่วคราว ติดป้ายที่มา + ความมั่นใจทุกช่อง
→
B · Provisional (เฟส 1)
สถานะ DOC_PROVISIONAL · เดินงานตามนโยบาย P0 หรือ P1 (เลือกรายเครื่อง)
→
C · Engineer Setup (เฟส 2)
วิศวกรหน้าเครื่อง อ่านค่าจริง เซ็ตพารามิเตอร์ รันเช็กลิสต์ + gate
→
D · Verified / Released
สถานะ ENGINEER_VERIFIED · เซ็นรับรอง → ผลิตจริงได้
แต่ละเครื่องเดินตามเส้นนี้อิสระ — เพิ่มเครื่องใหม่ได้เรื่อย ๆ โดยไม่ต้องรอเครื่องอื่นยืนยันเสร็จ งานเอกสาร/เตรียมโปรแกรม (CAD/CAM/nesting/simulation) เริ่มได้ทันทีตั้งแต่เฟส 1

2. นโยบายชั่วคราว — เลือกได้รายเครื่อง (P0 / P1)

ระหว่างที่เครื่องยังเป็น DOC_PROVISIONAL ให้กำหนดระดับที่อนุญาตต่อเครื่องในคอลัมน์ Interim ของ register

P0 ออฟฟิศ / เตรียมงานเท่านั้น

อนุญาต:

  • ออกแบบ CAD / CAM
  • จัด nesting layout, เขียนโปรแกรม offline
  • Simulation / ตรวจ envelope บนจอ
  • เตรียม tool list, ประเมินเวลา

ห้าม: ตัด/เจาะจริงบนเครื่องทุกกรณี · production = PROHIBITED

P1 ตัดแบบมีคนคุม + gate ครบ

อนุญาต (เฉพาะมีคนคุมตลอด):

  • ตัดจริงได้ ทีละงาน ถ้าผ่านครบ: simulation → dry-run/air-cut → first-article → มีคนรับงาน
  • ถือ envelope/tool/origin ว่า "ยังไม่ยืนยัน" ต้องเฝ้าจอ + feed hold พร้อมกด

ห้าม: เดินงานแบบไม่มีคนคุม · batch/night-run · ข้าม dry-run หรือ first-article

ห้ามเด็ดขาด (ทุก interim level): ตัดงานที่ generate จาก MONOLITH packet — NFP/NO_CUT บังคับด้วยกลไก verifier จนกว่า S17 ปิด + real-cut gate ผ่าน (CT-DEC-002 §11.6) · P1 ครอบเฉพาะงานกระบวนการเดิมของโรงงาน (ใบสั่งเดิม ไม่ผ่าน MONOLITH) เท่านั้น — ตาม ADR-065/ADR-070

ทั้ง P0 และ P1 ยังไม่ใช่การปล่อยผลิตจริง — manufacturing_release ยังเป็น PROHIBITED/CONDITIONAL จนกว่าวิศวกรจะยืนยัน (เฟส 2). P1 คือ "ตัดได้แบบระวังสูงใต้การควบคุมมนุษย์" ไม่ใช่ปล่อยเดินเอง

5. เช็กลิสต์วิศวกรหน้าเครื่อง (เฟส 2 — ใช้ซ้ำทุกเครื่อง)

วิศวกรทำครบทุกข้อ แล้วเซ็นรับรอง จึงเลื่อนเป็น ENGINEER_VERIFIED และปลด release

  1. Nameplate — ถ่ายรูปป้ายรุ่น + serial จดรหัสรุ่นเต็ม เทียบกับโปรไฟล์
  2. เวอร์ชัน About — NCstudio / HMI shell / รุ่นการ์ดคอนโทรล + Windows จดให้ตรงจริง
  3. Working envelope — ยืนยัน X/Y/Z จริง (jog สุดระยะ) เทียบกับค่าเอกสาร
  4. Import screen — ถ่ายหน้าเปิดไฟล์ ดูนามสกุลที่รับจริง freeze accepted_import_formats
  5. Tool table + Origin — จด T→ตำแหน่งแม็กกาซีน, tool length, วิธีเปลี่ยนทูล (M6/T), WCS G54 + มุม origin จริง + ทิศแกน
  6. File transfer + program path — จดช่องทางส่งไฟล์จริง + โฟลเดอร์ที่เครื่องรัน (redact ความลับ/รหัส)
  7. Known-good job — สร้าง+ล็อกงานอ้างอิง (immutable, เก็บ hash) รันผ่านจริง
  8. Gate: Simulation → Dry-run/Air-cut → First-article — ผ่านครบ วัดชิ้นแรกเข้า tolerance

6. เพิ่มเครื่องใหม่เข้าคิว deep-research

ส่งข้อมูลเท่าที่มีต่อ 1 เครื่อง เดี๋ยวผมยิง deep-research สร้างโปรไฟล์ชั่วคราวให้ (เหมือน KDT):

  • ยี่ห้อ + รุ่นเต็ม (รวม suffix ถ้ามี เช่น KN-2409LP)
  • ชนิดเครื่อง (nesting / router / beam saw / edge bander / boring ฯลฯ)
  • Controller ถ้ารู้ (NCstudio / Syntec / Osai / Fanuc / Beckhoff …) — ไม่รู้ก็ได้
  • เอกสาร/รูป ถ้ามี (สเปกชีต, รูปหน้าจอ, รูป nameplate) แนบมาได้
  • จำนวน + interim level ที่อยากได้ (P0 หรือ P1) ต่อเครื่อง
ส่งเป็นลิสต์ก็ได้ เช่น: "KDT KN-2412 ×2 (P0), Nanxing NB-9 ×1 (P1), Homag ... " ผมจะทำทีละเครื่องแล้วอัปเดตลง Fleet Register นี้ให้
