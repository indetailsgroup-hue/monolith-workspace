# CI Bypass Patterns - GATE SAFETY
# ================================
#
# These patterns are FORBIDDEN in production code.
# BLOCK = CI FAIL (exit code 1)
# WARN = Review required (exit code 2 unless --strict)
#
# Format: GATE|SEVERITY|PATTERN|DESCRIPTION
# - GATE: G9, G10, G10.1, G10.2, ALL
# - SEVERITY: BLOCK (always fail), WARN-HIGH, WARN-MED, WARN-LOW
#   - WARN-HIGH: Likely gate bypass, needs immediate attention
#   - WARN-MED: Quality/determinism issue, should fix before release
#   - WARN-LOW: Code hygiene, fix when convenient
# - PATTERN: grep -E compatible regex
# - DESCRIPTION: Human-readable explanation
#
# Lines starting with # are comments
# Empty lines are ignored
