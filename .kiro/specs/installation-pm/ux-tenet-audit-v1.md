# UX Tenet Audit — Installation PM (v1)
### "หน้าบ้านง่ายจนคนรัก หลังบ้านรับความยากไว้เอง"

> ตรวจ Installation PM spec ทั้งชุดด้วยไม้บรรทัด Governing UX Tenet (PRD §1) — หาจุดที่เผลอผลักความยากไปให้ผู้ใช้หน้างาน แล้วเสนอวิธีดันกลับเข้าหลังบ้าน
> **เกณฑ์วัด:** ต่อ 1 งาน วัด **(a) จำนวน tap/การกระทำ** และ **(b) จำนวนศัพท์ระบบที่ผู้ใช้เห็น** — ยิ่งน้อยยิ่งดี · เป้า: ช่างที่ไม่เคยอบรมทำจบได้เอง
> สถานะ: การแก้ที่ ✅ = apply แล้วในรอบนี้ · 📝 = บันทึกเป็นข้อกำหนดหลังบ้านให้ทำตอน implement

## วิธีให้คะแนน

| ระดับ | ความหมาย |
|-------|----------|
| 🟢 ผ่าน | ผู้ใช้ทำจบด้วยการกระทำธรรมชาติ (ถ่ายรูป/กดปุ่ม/พิมพ์สั้น) ไม่เห็นศัพท์ระบบ |
| 🟡 เสี่ยง | มีจุดที่ผู้ใช้ต้องจำ/พิมพ์/เลือกเกินจำเป็น — ดันเข้าหลังบ้านได้ |
| 🔴 leak | ผู้ใช้เห็นศัพท์ระบบ/ต้องกรอกฟอร์ม/ต้องรู้โครงสร้างภายใน |

## สรุปหลักที่ได้จาก audit (กติกาหลังบ้านถาวร)

1. **ไม่มี hashtag/command บังคับ** — ทุก action เริ่มจากการกระทำธรรมชาติ (ส่งรูป) แล้ว bot เสนอ**ปุ่ม**; hashtag เป็น shortcut ทางเลือกของ power user เท่านั้น
2. **ห้ามโชว์ id/key/ศัพท์โครงสร้าง** ให้ผู้ใช้หน้างาน — ห้อง = ชื่อภาษาคน ("ครัว"), งาน = "งานของคุณ", ไม่มี room_id/work_item_id/template_key/capture_type บนจอช่าง
3. **หลังบ้านเดาให้มากที่สุดก่อนถาม** — ห้อง/เลน/โปรเจกต์ infer จาก assignment + group binding; ถามเฉพาะเมื่อกำกวมจริง และถามด้วยปุ่มภาษาคน
4. **ทุก approval = Flex การ์ดเดียวในที่ที่คนอยู่แล้ว (LINE)** — ไม่ต้องสลับไปเปิด console/PWA
5. **default view ต่อบทบาท** — ช่างเห็น "งานของฉันวันนี้", หัวหน้าเห็นภาพรวมบ้าน, ลูกค้าเห็นเฉพาะที่ curated

## การแก้ที่ apply เข้า spec รอบนี้ ✅

- requirements Req 13.2 + line-arch §4: เปลี่ยน "ช่างพิมพ์ #ปัญหา/#ผูก" → **"ส่งรูป/เข้ากลุ่ม แล้ว bot เสนอปุ่ม; hashtag = ทางเลือก"**
- design D-11 + form-templates: เพิ่มกติกา **"ผู้ใช้หน้างานเห็นชื่อภาษาคนเท่านั้น ห้ามโชว์ id/key"** + **"หลังบ้าน infer ห้อง/เลนก่อนถาม"**
- tasks: เพิ่ม 1.6b (default view "งานของฉันวันนี้") + note บน 1.7/1.8 ว่าใช้ปุ่มไม่ใช่ command
