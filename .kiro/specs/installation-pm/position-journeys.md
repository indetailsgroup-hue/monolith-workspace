## Sales (H2 เจ้าหน้าที่ฝ่ายขาย · H1 ผจก.ขาย)

บทที่มีมติแล้ว (ADR-040/041/042): คุย lead ผ่าน OA Manager · กรอกใบบันทึกความต้องการ (Wave A + ส่ง customer_id) · ตั้งกลุ่มบ้าน + #ผูก · อยู่ในกลุ่มตลอดเส้น · curated updates ร่วมหัวหน้า · รับ referral lead ที่ tag ที่มา

> **มติ Sale-1 (owner, 7 ก.ค. 2026) — ราคาเบื้องต้น = ระบบคำนวณ "ช่วงราคา" จากใบ requirement**: ตารางเรทต่อ ตรม. ต่อเกรดวัสดุ (config โดย B4/PM — ไม่ hardcode) × design_scope_sqm → ช่วง min–max + disclaimer "ยืนยันหลังวัดจริง" ในการ์ด; Sale กดสร้างจากใบ requirement ใน 10 วินาที; **ทุกตัวเลขที่ส่งลูกค้า snapshot ลง audit** (ไม่มีราคาปากเปล่าลอยค้าง); ปัดตก: ประเมินนอกระบบ (ไม่มีร่องรอย/เผา lead) และบังคับผ่าน B4 ทุกครั้ง (คอขวด lead ร้อน)
> **มติ Sale-2 (owner) — ระบบตาม lead เงียบ**: lead ไม่มีใบ requirement/ยังไม่ปิด เงียบเกิน 3 วัน → push ส่วนตัวถึง Sale (น้ำเสียงเพื่อนเตือนเพื่อน Req 12.2) · เกิน 7 วัน → FYI ถึง H1 (ภาพรวม lead กำลังหลุด); ตัวเลข config + เคารพ quiet hours; **พ่วงปุ่ม "ปิด lead + เหตุผล"** (แพงไป/เงียบเอง/ไปเจ้าอื่น) — ของฟรีชิ้นใหม่: ข้อมูล lost-reason ให้ทีมการตลาด H3/H4; reuse sweep pattern + conversations.last_activity_at ที่มีอยู่; ปัดตก: รอ Phase 3 (lead หลุดเพราะลืมตาม = เงินหายเงียบสุด) และโพสต์กลุ่มทีม (ประจาน = เกลียดระบบ)
> **มติ Sale-3 (owner) — สัญญา generate จากระบบ + เซ็นกระดาษ→capture**: template สัญญา (ทนาย review ครั้งเดียว) เติมจากข้อมูลเดียวกับที่ระบบเดินงาน (ราคา B4 + scope ใบ requirement + งวด 50/30/15/5 + ประกัน 1 ปี) → PDF เข้ากลุ่ม → เซ็นกระดาษ (เจอกันตอนวัดอยู่แล้ว) → รูปเข้า capture `signed_contract` (immutable) → Sale mark signed → **การ์ดงวดมัดจำ 50% ยิงอัตโนมัติ**; หลัก: สัญญากับระบบไม่มีวันขัดกันเพราะมาจากข้อมูลก้อนเดียว; ปัดตก: สัญญานอกระบบ (คนละฉบับตอน dispute) และ e-signature (foreign SaaS — ADR-021 + เกินจำเป็นเมื่อเจอตัวจริง)
> **มติ Sale-4 (owner) — v1 = attribution + หน้าสรุปให้ H1, ไม่มี target/commission**: ทุกบ้านผูกเจ้าของ Sale จากคนกรอกใบ (เกิดเองแล้ว — ศูนย์การกรอกเพิ่ม) → สรุปให้ H1: สัญญา/เดือน ต่อ Sale + มูลค่า + lost-reason (จากปุ่มปิด lead); commission = มติแยกของ HR/การเงินเมื่อนโยบายนิ่ง (ระบบพร้อมรับเพราะ attribution วางแล้ว); ปัดตก: commission เต็มรูปตอนนี้ (นโยบายไม่นิ่ง + ชั้นความลับใหม่) และไม่มีเลย (H1 ใช้ Excel ทั้งที่ข้อมูลครบ = เสียของ)
> **มติ Sale-5 (owner) — หน้าแรก Sale = "งานขายของฉันวันนี้" 3 ส่วนเรียงตาม action**: ① Lead ต้องตาม (เงียบนานสุดก่อน + ปุ่มเปิดแชท/ปิด lead) ② บ้านที่กำลังเดิน — โชว์เฉพาะ*จุดที่รอ Sale* ("ยังไม่ตั้งกลุ่ม"/"สัญญายังไม่ mark signed"/"งวด 2 ค้าง 5 วัน") ③ ปุ่มใหญ่ "+ เปิดใบความต้องการใหม่"; สูตรเดียวกับ "งานของฉันวันนี้" ของช่าง — ระบบสแกนแล้วสรุปว่าคุณค้างอะไร; ปัดตก: list รวมแบบ office (Sale ต้องคิดเอง) และ dashboard ตัวเลข (ของ H1 ไม่ใช่คนวิ่งงาน)

**สรุปตำแหน่ง Sales: ครบ 5 มติ — งานระบบ = tasks Phase SJ (SJ-1…SJ-5)**
## D4 หัวหน้าทีมติดตั้ง (Installation Team Lead)

บทที่มีมติแล้ว: T0+approve เริ่ม · คุมทุกเลน · รับ #ปัญหา · ใบปิดบ้าน (RACI gate) · ส่งตรวจรับ · punch list · อยู่กลุ่มลูกค้า · จัดทีมโดย D3 (roster); JD: คุมเวลาเข้า-ออกไซต์ + รายงานทุกวัน + ความสะอาด/ปลอดภัย

> **มติ D4-1 (owner) — รองรับหลายบ้าน active ต่อหัวหน้าตั้งแต่แรก**: ความจริงมีช่วงซ้อน (บ้านเก่า punch ค้าง/รอตรวจรับ + บ้านใหม่เริ่ม T0); โครงข้อมูลรองรับแล้ว — งานคือฝั่งประสบการณ์: **ทุกแจ้งเตือน tag ชื่อบ้านเสมอ** (กติกาเดียวกับกลุ่มโรงงาน ADR-041) + หน้าจอ "ทุกบ้านของฉัน" เรียงเร่งด่วน (#ปัญหา เปิด > รอ T0 > punch ค้าง > ปกติ); ปัดตก: จำกัด 1 บ้าน (ฝืนจริง = คนอ้อมระบบ) และไม่ออกแบบรองรับ (แจ้งเตือนไม่ tag บ้าน = บั๊ก UX รอเกิด)
> **มติ D4-2 (owner) — เวลาเข้า-ออกไซต์ = หัวหน้ากดคนเดียว รวมทีมต่อบ้านต่อวัน**: เช้ากด "เข้างาน" + ติ๊กใครมา (รายชื่อจาก roster J2.5) · เย็นกด "เลิกงาน"; ช่างไม่ต้องทำอะไร (UX tenet — ภาระอยู่ที่คนที่ JD กำหนด); **ชั่วโมง×คน ต่อบ้าน → คูณเรทเข้า Job Cost (PK-2) อัตโนมัติ = P&L ต่อบ้านได้ฝั่งแรงงานจริง**; D3/D2 เห็นเวลาเข้า-ออกทุกทีมตาม JD; ปัดตก: ช่างกดเอง+GPS (friction 15 คน×2/วัน + PDPA พนักงาน) และไม่เก็บ (P&L โกหกครึ่งใบ); ยกระดับรายคนได้ทีหลังถ้าจำเป็น
> **มติ D4-3 (owner) — รายงานประจำวัน = ระบบร่างเอง หัวหน้าเติมหมายเหตุ+กดส่ง**: การติ๊ก checklist + ส่งรูปของช่าง = คือรายงานของช่างในตัว (JD ถูกทำผ่านงานปกติ); เย็นระบบประกอบ (ความคืบหน้าต่อเลน + รูป + #ปัญหา + ชั่วโมงทีมจาก D4-2) → หัวหน้าเติม 1–2 บรรทัด → ส่งถึง D3/D2/PM + PDF (task 1.6); ภาระ 20 นาที → 90 วินาที + เทียบได้ทุกบ้านทุกวัน; หมายเหตุภายในไม่ไปลูกค้า (curated แยกเส้น); ปัดตก: ช่าง 15 คนพิมพ์เอง (เกลียดภายในสัปดาห์) และ free-form (เทียบไม่ได้)
> **มติ D4-4 (owner) — escalation = ประตูเดียว (issue เดิม) + ประเภท 4 ปุ่ม → ระบบ route เอง**: ของขาด/ผิด → E6+E2+E7 · ติดตั้งตามแบบไม่ได้ → B2+B4 · **ลูกค้าขอเพิ่ม/แก้หน้างาน → Sale+PM + เข้าเส้น scope change/requote (ADR-037) — ห้ามตกลงปากเปล่า** (ปิดรูรั่วคลาสสิก: งานเพิ่มที่ไม่เคยถูกคิดเงิน) · ความปลอดภัย → D3+HSE ทันที ข้าม quiet hours; ทุกประเภท SLA ไม่มีคนรับ → ไต่ D1 PM อัตโนมัติ (ตรง JD); implement = category+routing map บน installation_issues เดิม; หลัก D-12: หัวหน้าไม่ต้องจำว่าเรื่องไหนโทรหาใคร — ระบบจำแทน
> **มติ D4-5 (owner) — หน้าแรกหัวหน้า = "บ้านของฉันทั้งหมด" + ปุ่มถัดไปที่ถูกต้องต่อบ้าน**: การ์ดต่อบ้านเรียงเร่งด่วน (D4-1) + ปุ่มเดียวตามสถานะ/เวลา (เช้า→เข้างาน · เลนครบ+รูปครบ→ส่งปิดบ้าน · เย็น→ส่งรายงาน · ปัญหาใหม่→ดูปัญหา); logic state→action ตารางเดียวหลังบ้าน; ปรัชญาเดียวทั้งแอป: เปิดมา 3 วินาทีรู้ว่ากดอะไร

**สรุปตำแหน่ง D4: ครบ 5 มติ — งานระบบ = tasks Phase DJ (DJ-1…DJ-4)**
